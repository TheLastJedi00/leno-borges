Estudo de Caso: BF Academy

Decisões de arquitetura por trás de uma plataforma EdTech.

Do Gargalo à Visão Escalável

O desafio inicial era claro: uma escola de idiomas estava limitada por um processo manual de criação de conteúdo que levava até 7 dias. O objetivo de negócio era: automatizar essa criação de conteúdo possibilitando um teto de alunos muito maior em uma plataforma capaz de escalar para vários de usuários com custo operacional mínimo eliminando qualquer gargalo por operação manual.

Estratégia de Backend: Por que NestJS ao invés de Java/Spring?

Com sólido conhecimento em Java, meu padrão seria uma arquitetura robusta com Spring Boot. No entanto, um bom arquiteto escolhe a ferramenta certa para o trabalho, não apenas a mais familiar. Todo o backend da BF Academy foi projetado para rodar em uma infraestrutura serverless para gerenciar custos e escalar sob demanda.

Nesse cenário, a latência de "cold start" da JVM é um gargalo de performance significativo. O ecossistema Node.js, especificamente o NestJS, ofereceu tempos de inicialização quase instantâneos e um consumo de memória muito menor, tornando-se a escolha de engenharia clara.

Crucialmente, o NestJS me permitiu manter os padrões de design de alto nível que valorizo no ecossistema Spring, incluindo Injeção de Dependência, uma estrutura modular e uma abordagem declarativa usando Decorators. Foi o melhor de dois mundos: performance para a plataforma e estrutura para o código.

Frontend: A Escolha Deliberada pelo Angular

Para um Learning Management System (LMS) complexo e rico em funcionalidades, um framework opinativo fornece as diretrizes necessárias para a manutenibilidade a longo prazo. Angular foi a escolha deliberada. Sua implementação estrita do TypeScript garante a segurança de tipos em escala, sua arquitetura baseada em componentes promove a reutilização e, mais importante, o poder do RxJS é inigualável para gerenciar os fluxos de dados assíncronos e complexos inerentes a uma plataforma de aprendizado moderna — desde o acompanhamento de progresso em tempo real até os estados de exercícios interativos.

Hack de Infraestrutura: O Player de Vídeo Custo-Efetivo

Um grande obstáculo para qualquer startup baseada em vídeo é o custo proibitivo de largura de banda e armazenamento. Serviços como AWS S3 e CloudFront podem esgotar rapidamente um orçamento. A solução foi um "hack" de infraestrutura não convencional: construímos uma camada de abstração que utiliza a hospedagem de vídeo gratuita e de classe mundial do YouTube para vídeos não listados. O frontend renderiza esses vídeos em um player totalmente customizado e sem a interface do YouTube, que busca o stream de vídeo enquanto ofusca o link direto. Isso proporcionou uma rede de entrega de vídeo segura e de alta performance com custo de infraestrutura virtualmente zero.

Arquitetura e Qualidade: Clean Architecture & DDD

O núcleo da aplicação — as regras pedagógicas, a lógica de progresso do aluno e a estrutura de conteúdo — foi construído seguindo os princípios de Clean Architecture e Domain-Driven Design (DDD). Isso significa que o "Core do Negócio" está completamente isolado de preocupações externas, como bancos de dados ou APIs. O benefício direto é a flexibilidade; se precisarmos migrar do Firebase para um banco de dados PostgreSQL amanhã, as entidades e os casos de uso críticos para o negócio permanecem intocados. A infraestrutura é um detalhe, não a base.

IA Generativa: O Motor de Estruturação de Conteúdo

Nosso uso de IA Generativa vai além de um simples chatbot. Integramos a Google Gemini API como um poderoso motor de estruturação de conteúdo. Especialistas pedagógicos fornecem inputs de alto nível (por exemplo, um tópico, uma lista de vocabulário, uma regra gramatical), e o papel da IA é transformar isso em um esquema JSON estritamente validado. Esses dados estruturados são então consumidos pelo frontend Angular para renderizar dinamicamente uma rica variedade de exercícios interativos. Não é apenas criação de conteúdo; é design instrucional automatizado.
